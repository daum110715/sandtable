# AI 治理

> 状态：基线已确认（2026-07-12）。

## 项目所有者决策

- D017 高影响行动与人工审批范围：世界内行动由规则裁决；修改史料、规则、权限、模型和公开内容需人工审批。
- D018 工具、网络与文件权限：默认拒绝任意网络、Shell 和文件访问，只开放领域工具允许列表。
- D019 隐私与会话保留：数据最小化；原始模型会话保留 30 天，审计元数据保留 180 天，支持删除与导出。
- D020 敏感历史内容政策：史实/模拟显式区分，敏感内容分级，公开内容自动检测加人工复核。

## 信任模型

模型是不可信但有能力的解释者和提议者。它可以协助理解 Root 改写指令，也可以让历史行动者形成计划和行动意图；它不能自行获得 Root 权限、声明史实或直接修改世界状态。

## 权限层级

1. **只读观察**：读取行动者在当前模拟时刻可知的信息。
2. **受控提议**：通过工具能力提交结构化行动意图。
3. **高影响提议**：战争、处决、政权变更等需额外规则或人工复核。
4. **Root 改写**：用户可向具体世界线下达改写指令，但不能借此修改全局史料、规则、模型或权限。
5. **系统管理**：修改基线、规则、模型或权限，只能由项目管理流程执行，不开放给模型或世界内行动者。

## 必须执行的控制

- 工具能力采用允许列表，默认拒绝未声明能力。
- 使用 Pi 的工具调用前置钩子进行预检，但最终权限判断由 Sandtable 治理模块所有。
- 模型上下文只包含行动者合法观察，防止“上帝视角”泄漏。
- 结构化输出先经过语义验证，再进入裁决。
- 提示词、模型、工具和规则都带版本，运行记录可追溯。
- 外部网络、文件系统和进程访问默认关闭；若未来启用，应放入独立沙盒并记录 ADR。

## 历史可信度

- 史料主张必须携带来源，冲突来源可以并存。
- AI 对史料的摘要是解释，不自动成为新的史料主张。
- 不确定内容必须标明推断等级，不使用确定语气掩盖资料缺口。
- 对仍有现实伤害风险的民族、宗教、战争和人物议题，应保留审阅与内容治理机制。

## 评估维度

- **遵循观察**：行动者是否使用了其不应知道的信息。
- **行动有效性**：行动意图是否满足结构和领域约束。
- **人物一致性**：行为是否与已声明的目标、关系和历史条件相容。
- **世界一致性**：裁决是否维护资源、时间和因果不变量。
- **解释忠实度**：展示给用户的理由是否来自真实裁决依据。
- **模型可替换性**：不同模型是否都能通过同一行动意图 interface 参与。
