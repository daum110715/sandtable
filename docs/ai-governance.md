# AI 治理

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D008、D017、D018、D030、D031。关联 ADR：ADR-0006。

## 项目所有者决策

- D008 AI 与世界规则的权责：1.0 由演员 Agent 实时推演结果，记录员 Agent 写回结构化状态；确定性规则裁决与裁判 Agent 留待后续。
- D017 高影响行动与审批：1.0 用户拥有完整权限，世界内行动由 AI 推演产生；修改系统级配置需人工审批。
- D018 Agent 工具与外部权限：Agent 默认无任意网络/Shell/文件权限，仅可读写世界状态数据库与事件日志。
- D030 1.0 权限设计：用户拥有完整权限，不设限制，不做迷雾/代价。
- D031 多 Agent 分工：演员/记录员/裁判（可选）。

## 信任模型

模型是推演的执行者：演员 Agent 读取世界状态生成推演，记录员 Agent 把状态变更写回世界状态数据库。1.0 不设规则裁决模块；裁判 Agent 可选，默认不启用。世界状态的写入只经记录员 Agent，演员 Agent 不直接写状态。

## 权限（1.0）

1. **改写者（用户）**：拥有完整权限，可自由输入任意改写；1.0 不设分层、不做迷雾/代价。
2. **演员 Agent**：只读世界状态，生成推演；不直接写世界状态。
3. **记录员 Agent**：把推演结果拆解为结构化状态变更，写回世界状态数据库与事件日志。
4. **裁判 Agent**：可选/后期，检查矛盾；1.0 默认不启用。
5. **系统管理**：修改模型、提示词、Agent 配置或世界状态 schema 只能由项目管理流程执行，不开放给 Agent。

## 必须执行的控制（1.0 最小）

- Agent 默认无任意网络、Shell、进程和文件系统访问；仅可读写世界状态数据库与事件日志（D018）。
- 只有记录员 Agent 能写回世界状态数据库；演员 Agent 无写入路径。
- 模型上下文只包含当前世界状态与本次改写，世界状态以结构化数据为准，不依赖对话记忆堆叠。
- 提示词、模型与 Agent 配置带版本，运行记录可追溯。
- 修改系统级配置（模型、提示词、权限、schema）需人工审批（D017）。
- 每次模型运行设置 Token、时间与重试预算，可随时取消。

## 推演可信度（1.0）

- 推演内容不得伪装为史实（D010 原则保留；具体 UI 标识留待后续）。
- 不确定内容应避免使用确定语气掩盖资料缺口。
- 对仍有现实伤害风险的民族、宗教、战争和人物议题，应保留基础审阅与内容治理机制（细化留待后续，D020）。

## 评估维度（1.0）

- **状态一致性**：推演后世界状态是否与前序矛盾。
- **推演合理性**：推演是否基于当前世界状态与本次改写。
- **写回忠实度**：记录员 Agent 的结构化变更是否与推演叙事一致。
- **改写自由度**：用户能否输入任意颗粒度的改写。
- **可回溯性**：事件日志能否按顺序还原改写-推演历史。
- **模型可替换性**：不同模型是否都能通过同一 Agent 协议参与（D014 待定）。
