# 数据与存储

> 状态：1.0 方向已确认；D016 已确认为 SQLite（2026-07-13，ADR-0007）。关联决策：D009、D018。

## 数据权威性

| 类别 | 权威来源 | 保留与删除 |
| --- | --- | --- |
| 场景初始世界状态 | 预设场景定义 / 自定义背景 | 长期保留；预设内容修订不覆盖已被引用的初始状态 |
| 世界状态 | 世界状态数据库 | 唯一权威；由记录员 Agent 写回 |
| 事件日志 | 事件日志存储 | 不可原地修改；按顺序保留，支持回溯 |
| 对话上下文 | 对话记录 | 仅作输入载体，不是世界状态权威；可短期保留 |
| Agent 原始运行记录 | 模型运行记录 | 1.0 保留策略待定（D019） |

## 存储基线（D016）

- 数据存储技术：**SQLite**（1.0）。运行时：Node 内置 `node:sqlite`；默认文件路径在项目目录（如 `data/sandtable.sqlite`），可用环境变量覆盖。
- 须满足：
  - 世界状态以结构化数据持久化，可按要素（人物、势力、资源、地理、关系等）查询与更新（1.0 以 `WorldState` JSON 快照 + 领域层查询为主）。
  - 事件日志为只追加、不可原地修改、可按顺序读取。
  - 世界状态写入与对应事件追加应在同一事务内，避免状态与日志不一致。
- 1.0 不引入独立搜索系统；只有测量证明不足时再考虑。
- 内存 adapter 保留用于单元测试与无库路径。

## 一致性与事务（1.0）

- 记录员 Agent 的结构化状态变更与事件日志追加位于同一事务。
- 提交改写使用幂等键，网络重试不得重复产生事件。
- 1.0 单人单会话，按顺序处理，不涉及并发冲突（D012）。
- 事件一旦追加不可原地修改；修正通过新事件表达，不回写旧事件。

## 版本与迁移（1.0 最小）

- 事件日志、世界状态 schema、模型与 Agent 提示词分别可版本化。
- 1.0 不做版本化史料基线（D005 后续迭代）。
- 破坏性 schema 迁移优先向上兼容读取层；无法兼容时冻结旧会话并提供导出。

## 待定项

- 世界状态数据库是否从 JSON 快照演进为规范化表（1.0 非必须）。
- Agent 原始运行记录与会话保留策略（D019）。
