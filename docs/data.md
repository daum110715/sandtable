# 数据与存储

> 状态：基线已确认（2026-07-12）。关联决策：D009、D013、D016、D019。

## 数据权威性

| 类别 | 权威来源 | 保留与删除 |
| --- | --- | --- |
| 史料与来源 | 版本化史料目录 | 长期保留；修订不覆盖旧版本 |
| 历史基线 | 已批准的基线版本 | 长期保留；已被世界线引用后不可原地改写 |
| 世界线事件 | PostgreSQL 事件账本 | 权威记录；用户删除时按政策删除或匿名化 |
| 世界状态投影 | 从事件重建 | 可清除并重建 |
| Agent 原始会话 | 模型运行记录 | 默认 30 天，之后删除或去标识化 |
| 审计元数据 | 工具、规则、模型和权限记录 | 默认 180 天 |
| 用户账户 | 身份提供方 + 本地用户映射 | 用户可导出和删除 |
| 媒体与导出物 | S3 兼容对象存储 | 随所属世界线生命周期管理 |

## 存储基线

- PostgreSQL：账户映射、史料目录、基线、世界线、事件、投影、规则版本和审计索引。
- Redis：后台任务队列、短期缓存、速率限制和短租约；不存放不可恢复的权威数据。
- S3 兼容对象存储：地图、来源附件、世界线导出和分享图。
- 搜索初期使用 PostgreSQL 全文搜索与索引；只有测量证明不足时再引入独立搜索系统。

## 一致性与事务

- 追加事件和推进世界线游标位于同一数据库事务。
- 命令使用幂等键，网络重试不得重复产生事件。
- 每条世界线采用乐观并发版本号；冲突返回重新读取和再裁决。
- 队列采用至少一次投递，处理器必须幂等。
- 投影失败不回滚已提交事件，通过检查点重新构建。

## 版本与迁移

- 事件模式、规则、历史基线、模型、提示词和工具清单分别版本化。
- 世界线固定其基线和规则版本，不静默升级。
- 破坏性事件迁移优先使用向上转换读取层；无法兼容时冻结旧线并提供导出。
- 每日自动备份数据库，定期验证点时间恢复；对象存储启用版本和生命周期策略。
