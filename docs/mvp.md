## 目标

用一个小型历史切片证明完整闭环：历史基线 → 行动者观察 → Pi 认知回合 → 行动意图 → 裁决 → 事件 → 世界状态 → 分支与回放。

## 建议切片约束

- 单一地区与较短时间跨度；
- 3 至 5 个关键行动者；
- 1 至 2 个明确制度；
- 少量可量化资源；
- 一个历史关键节点；
- 至少两种有意义但可解释的分支结果。

具体年代与事件仍为待定，应根据史料可获得性、敏感性和规则可表达性选择，而不是只看知名度。

## MVP 必须证明

- 可以从带来源的历史基线启动世界线。
- 每个行动者只能读取自己的观察。
- Pi 能稳定产生符合协议的行动意图。
- 无效行动会被裁决拒绝且理由清楚。
- 有效行动生成事件，世界状态只能由事件投影更新。
- 用户可以在关键事件前创建分支并比较结果。
- 在不重新调用模型的情况下重放已记录世界线。
- 用户界面能区分史实、模拟和不确定推断。

## MVP 不包含

- 全球历史数据库；
- 数千行动者并发；
- 开放世界自由文本直接改写规则；
- 多人实时对战；
- 经济、军事、外交等所有领域的高保真统一模型；
- 自动发表“历史结论”。

## 进入实现阶段的闸门

在创建业务代码前，应完成并确认：

1. MVP 历史切片及史料来源清单；
2. 行动意图与事件的语义协议；
3. 首批裁决规则和不变量；
4. Pi 接入方式（SDK 或 RPC）及版本策略；
5. 数据持久化与重放验证方案；
6. 内容治理和人工复核范围；
7. 验收场景与失败标准。
