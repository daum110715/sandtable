# 1.0 范围

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D006、D032。目标：做出 1.0，不做过度设计。

## 目标

用最小闭环证明完整体验：自定义世界设定 → 自由输入改写 → AI 根据世界状态推演 → 状态一致性有保障 → 继续输入循环。

## 1.0 必须跑通

- 用户可输入世界设定，创建通用初始世界状态。
- 用户可在对话中自由输入任意颗粒度的改写（不是选择题）。
- 演员 Agent 读取当前世界状态生成推演；记录员 Agent 把状态变更写回世界状态数据库。
- 世界状态以结构化数据库为权威，不依赖对话上下文记忆。
- 每次改写与推演作为独立事件追加到事件日志，可按顺序回溯。
- 上帝面板常驻展示当前世界结构化状态。
- 推演后世界状态结构化更新，且不与前序状态矛盾。

## 1.0 不包含

- 上帝面板的细节展示内容（仅作最小结构化展示）。
- 预设场景库与题材限制。
- 权限分层、迷雾、代价机制（1.0 用户拥有完整权限）。
- 确定性重放、规则裁决引擎、裁判 Agent（1.0 默认不启用）。
- 显式世界线分支与比较。
- 账户同步、多人、分享。
- 版本化史料基线与来源置信度体系。
- 商业化与部署。

## 进入实现阶段的闸门

在创建业务代码前，应完成并确认：

1. 1.0 世界设定的最小数据结构；
2. 世界状态数据库与事件日志的最小 schema；
3. 演员/记录员 Agent 的输入输出协议；
4. Agent 框架与模型供应商选择（D007、D014、D015）；
5. 数据存储技术选择（D016）；
6. 1.0 验收场景与失败标准。

## 1.0 验收门槛（D021）

- 自由输入改写 → AI 合理推演 → 状态一致性（事件日志可回溯、世界状态不矛盾）全部通过才可发布。
- 替换模型或 Agent 框架不改变世界状态的数据语义。
