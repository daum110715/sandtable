# 质量与验收

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D021、D032。

## 1.0 发布级质量目标

- **改写自由度**：用户可自由输入任意颗粒度的改写，不被选项限制。
- **推演合理性**：AI 推演基于当前世界状态与本次改写，叙事合理。
- **状态一致性**：推演后世界状态结构化更新，且不与前序状态矛盾。
- **可回溯性**：事件日志可按顺序还原每次改写-推演。
- **状态权威性**：世界状态以结构化数据库为准，不依赖对话上下文记忆。
- **面板同步**：上帝面板能反映当前世界结构化状态。
- **可靠性**：模型超时、断网、重复提交和 Worker 重启不破坏世界状态与事件日志一致性。

## 验收矩阵（1.0）

| 层级     | 方法                                              | 发布标准                                         |
| -------- | ------------------------------------------------- | ------------------------------------------------ |
| 文档     | 链接、术语、决策与 ADR 一致性检查                 | 无未解释冲突，无核心空白                         |
| 领域协议 | 世界状态与事件日志 schema、Agent 输入输出协议测试 | 核心不变量 100% 通过                             |
| Agent    | 固定场景、状态一致性、写回忠实度评估              | 推演后世界状态无已知矛盾；记录员写回与叙事一致   |
| 世界状态 | 改写-推演循环、回溯测试                           | 事件日志可按顺序回溯；状态写入与事件追加事务一致 |
| 系统     | 重复提交、超时、断网、Worker 重启                 | 不产生重复事件；不破坏状态一致性                 |

## 评估资料（1.0 最小）

- 至少一个预设场景的初始世界状态。
- 一组覆盖不同颗粒度的改写样例（含“那天江上刮西北风”级细节）。
- 至少数条完整改写-推演循环及对应事件日志与状态快照。
- 覆盖模型拒绝、超时、重复提交和断网的故障集。

## 发布闸门（D021）

- 自由输入改写 → AI 合理推演 → 状态一致性（事件日志可回溯、世界状态不矛盾）全部通过才可发布。
- 替换模型或 Agent 框架不改变世界状态的数据语义。
- P0/P1 缺陷、高危安全问题、数据不可恢复风险和史实/推演混淆均阻断发布。
- 发布由项目所有者签字。

## 后续迭代（不在 1.0）

- 确定性重放、规则裁决、权限分层、迷雾机制、分支与比较、账户与分享的相关验收标准留待后续阶段补充。
