# 质量与验收

> 状态：基线已确认（2026-07-12）。关联决策：D021。

## 发布级质量目标

- 历史准确性：所有关键基线主张有来源、版本和置信度，争议不被隐藏。
- 世界一致性：资源、时间、观察权限和制度不变量无已知阻断缺陷。
- 行动者可信度：不得使用不可见信息，行动意图符合其目标和约束。
- 解释忠实度：用户看到的原因来自真实裁决记录，不由模型事后编造。
- 可重放性：固定基线、规则、事件输入和随机种子时，关键状态哈希完全一致。
- 移动体验：320px 宽起可用，核心旅程无横向溢出，触控与可访问性达到既定标准。
- 可靠性：模型超时、断网、重复提交和 Worker 重启不会破坏世界线。

| 层级 | 方法 | 发布标准 |

| 移动体验 | 真机 + 自动化视口 + 屏幕阅读器检查 | 核心流程全部完成；WCAG 2.2 AA 阻断项为 0 |
| 系统 | 负载、故障注入、恢复和安全测试 | 达成 SLO；无高危安全缺陷 |

## 评估资料

- 固定赤壁历史基线及其来源快照。
- 至少 20 条黄金行动场景、20 条无效行动、10 条观察泄漏诱导。
- 至少 5 条完整世界线及每个检查点的状态哈希。
- 覆盖模型拒绝、超时、工具失败、重复消息、乱序和重试的故障集。
- 涉及战争、死亡、族群和仇恨内容的安全评估集。

## 发布闸门

- 所有核心不变量、确定性重放、权限隔离和来源追溯测试必须通过。
- P0/P1 缺陷、高危安全问题、数据不可恢复风险和史实/模拟混淆均阻断发布。
- 模型质量下降可以通过路由降级，但不能降低规则与权限门槛。
- 发布由项目所有者签字；历史内容变更同时需要历史审阅记录。
