# 文档协作指南

## 当前允许的贡献

- 补充背景、术语、候选方案和影响分析；
- 增加文档模板、图示和验证场景；
- 修正链接、错别字和内部矛盾；
- 不创建具体业务实现。

## 决策规则

- 项目所有者已于 2026-07-12 按 1.0 方向确认基线；只有项目所有者可以改变已确认决策或把后续 ADR 改为 `Accepted`。
- 其他贡献者不得把候选方案改写成既定事实。
- `待定`项必须保留 `决策：________` 形式，可列候选项与影响，不得代填答案。
- 新决策点先登记到 `docs/decision-register.md`。
- 已确认决策的变更必须保留理由和历史记录；难逆转的变更新增 ADR 并标记旧 ADR 为 `Superseded`。

## 文档变更检查

- 是否使用 `CONTEXT.md` 的统一术语？
- 是否新增了未登记的决策？
- 是否把候选项误写成结论？
- 是否与 ADR 冲突？
- 是否补充了验证方法和失败条件？
- 是否仍遵守禁止具体代码实现的限制（除非项目所有者明确解除）？
