# 产品愿景

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D001、D003、D028、D030、D032。

## 项目所有者决策

- D001 核心目标用户：喜爱历史假设与推演的互动用户；次要用户为历史教育者与内容创作者。
- D003 用户身份与权限：用户是拥有完整权限的改写者；1.0 不设权限分层、不做迷雾/代价。
- D028 核心交互 loop：选择场景 → 与 AI 自由对话输入任意改写 → AI 根据世界状态推演 → 继续输入，循环往复；不是选择题。
- D030 1.0 权限设计：用户拥有完整权限，不设限制。
- D032 1.0 优先级：做出 1.0，不做过度设计。

## 一句话

让用户自由改写历史事件，由 AI 根据结构化世界状态实时推演后续走向。

## 用户价值

历史模拟推演AI 不追求“让模型自由续写故事”，而是提供三种可感知的体验：

1. 用自由对话任意改写历史，细到“那天江上刮西北风”的颗粒度，不受选项限制。
2. AI 根据当前世界状态推演后果，而不是凭对话记忆随口编造。
3. 通过上帝面板和事件日志看见当前世界结构化状态与每次改写-推演的来龙去脉。

## 产品原则

- **改写自由**：用户可输入任意颗粒度的改写，不是选择题；1.0 拥有完整权限。
- **状态为权威**：世界状态以结构化数据库为准，不依赖对话上下文记忆。
- **推演有依据**：AI 推演读取当前世界状态，结果可追溯到本次改写。
- **变更可回溯**：每次改写与推演作为独立事件追加到事件日志。
- **不做过度设计**：1.0 只跑通自由输入 → AI 推演 → 状态一致性；面板细节、历史范围、权限分层、迷雾机制留待后续。

## 非目标

- 不做无世界状态的纯角色聊天。
- 不在 1.0 做确定性重放、规则裁决引擎、权限分层、迷雾/代价机制。
- 不把推演结果宣传为史实或历史预测。
- 不在 1.0 覆盖全部历史范围。

## 成功信号

- 用户能自由输入任意改写，AI 给出合理且不与前序矛盾的推演。
- 推演后世界状态结构化更新，事件日志可按顺序回溯。
- 上帝面板能反映当前世界结构化状态。
- 替换模型或 Agent 框架不改变世界状态的数据语义。
- 1.0 范围内的功能最小且闭环可用。
