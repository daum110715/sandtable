# 自定义世界设定，不提供预设场景

## Status

Accepted — 2026-07-13

Supersedes: D001、D004、D027、D028 中关于历史题材、预设场景及自定义历史背景的部分。

## Context

原型入口同时提供赤壁预设和“自定义背景”，但后者仍复用赤壁世界状态，仅将文字留在客户端会话中。这既限制了题材，也使 Agent 不能将用户设定作为权威上下文读取。

## Decision

- 1.0 不提供任何预设场景。
- 用户必须输入世界设定；名称可选，设定说明必填。
- 系统以该设定创建空白、通用的初始世界状态，不注入特定人物、时代、事件或规则。
- 世界设定随世界状态持久化，并传入演员 Agent 与记录员 Agent；浏览器会话只保留恢复界面的副本。
- 改写与推演交互、事件日志追加、状态权威和 Agent 分工保持不变。

## Consequences

- 正面：题材不受限；用户设定成为可追溯的权威输入；产品不再暗含某一历史叙事。
- 负面：空白世界的具体人物、势力和关系依赖模型/记录员建立；无模型的桩模式以通用「世界变化」资源推进状态，用于验证完整闭环。
- 兼容：旧的持久化快照可以缺少世界设定字段；新建世界一律写入。
