# AI 实时推演 + 结构化世界状态

1.0 采用多 Agent 推演架构：演员 Agent 读取世界状态生成叙事推演，记录员 Agent 把推演结果拆解为结构化状态变更写回世界状态数据库，每次改写与推演结果作为独立事件追加到事件日志。世界状态以结构化数据库为唯一权威，不依赖对话上下文记忆；裁判 Agent 可选，1.0 默认不启用。该决策取代旧架构 ADR-0001（Pi 作为 Agent 地基）与 ADR-0003（AI 提议、规则裁决），后者与“自由对话式 AI 实时推演”方向冲突。

## Status

Accepted — 2026-07-12

Supersedes: ADR-0001, ADR-0003

## Context

项目所有者于 2026-07-12 确认 1.0 方向为“用户自由改写历史事件 → AI 实时推演后续走向 → 状态一致性有保障”，并明确不做过度设计。旧架构以 Pi 行动者认知（ADR-0001）和“AI 提议、规则裁决”（ADR-0003）为核心，依赖确定性规则裁决、行动意图协议和离散事件调度，实现路径长、与自由对话式交互不匹配。1.0 只需跑通自由输入 → AI 推演 → 状态一致性。

## Decision

- 交互以自由对话进行：用户输入任意颗粒度的改写，AI 根据当前世界状态推演结果，循环往复；不采用选择题或结构化指令确认流程。
- 推演由演员 Agent 完成，直接产出叙事结果与状态变更，不再经独立规则裁决模块。
- 记录员 Agent 负责把推演结果拆解为结构化状态变更并写回世界状态数据库，保证世界状态一致、可追溯。
- 世界状态以结构化数据库为唯一权威，不依赖对话上下文记忆。
- 每次改写与推演结果作为独立事件追加到事件日志，支持历史回溯。
- 裁判 Agent（矛盾检查）为可选，1.0 默认不启用。
- 1.0 为单条演进世界线；显式世界线分支与确定性重放留待后续。

## Reason

1.0 优先以最短路径跑通“自由输入 → AI 推演 → 状态一致性”。结构化世界状态数据库 + 事件日志足以在 1.0 保证可追溯与一致性；规则裁决引擎、确定性重放、权限分层与迷雾机制属于过度设计，留待后续按真实需要迭代。

## Consequences

- 正面：交互自由度高、实现路径短、符合 1.0 范围与“不做过度设计”原则。
- 负面：放弃确定性重放与规则裁决的可解释性；状态一致性依赖记录员 Agent 的结构化写回质量；同一输入不保证产生相同结果。
- 后续工作：裁判 Agent、确定性重放、规则裁决、显式分支、权限分层、迷雾/代价机制均留待后续迭代，届时新增 ADR。
- Agent 框架（Pi 或其他）与模型供应商策略现为待定决策（D007、D014、D015），不在此 ADR 锁定。
