# 用户是拥有 Root 权限的历史改写者

用户不扮演单个历史行动者，而是从任意模拟时刻下达改写指令。系统为每次改写创建新世界线，把指令转换为明确标记的干预事件，再由受有限观察和制度约束的 AI 行动者及规则模拟后续影响；这样保留自由改写能力，同时保持史实分离、事件权威、回放和审计。

## Status

Accepted — 2026-07-12

## Consequences

- 改写指令不因“不符合历史”被拒绝，但含义不清、超出可表达领域或违反安全政策时必须要求澄清或说明限制。
- 改写不能原地修改父世界线，必须创建分支并记录干预事件。
- AI 行动者没有 Root 权限，仍只能提交行动意图。
- UI 的主要操作从“选择人物行动”改为“选择改写点并下达指令”。
- 系统必须展示指令的结构化解释，让用户确认后再开始模拟。
