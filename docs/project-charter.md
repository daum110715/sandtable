# 项目章程

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D001、D024、D026、D032。

## 项目身份

- 名称：历史模拟推演AI（暂定，最终名称待定，D026）
- 方向：让用户自由改写历史事件、由 AI 实时推演后续走向的互动模拟产品
- 核心端：Web（具体平台优先级待定，D002）；现有原型为 React + Vite
- Agent 地基：待定（D007）；1.0 采用演员/记录员/裁判多 Agent 分工
- 当前阶段：技术原型，目标做出 1.0

## 项目使命

让用户从一个历史场景出发，自由输入任意改写，由 AI 根据结构化世界状态实时推演后续走向，并通过世界状态数据库与事件日志保证推演可追溯、世界状态一致。

## 问题陈述

- 历史“假设”类讨论缺乏稳定的世界状态：通用 AI 聊天容易随对话漂移、自相矛盾，记不住“现在世界是什么样”。
- 传统策略游戏有规则但拘泥于选项，无法接受“那天江上刮西北风”这种自由颗粒度的改写。
- 历史模拟推演AI 把自由对话式改写、AI 实时推演与结构化世界状态组合为统一体验。

## 目标

- 跑通 1.0 闭环：自由输入改写 → AI 合理推演 → 状态一致性有保障。
- 世界状态以结构化数据库为权威，不依赖对话上下文记忆。
- 每次改写与推演作为独立事件追加到事件日志，可回溯。
- 上帝面板常驻展示当前世界结构化状态。
- 不做过度设计。

## 非目标

- 不在 1.0 做面板细节、历史范围锁定、权限分层、迷雾/代价机制、确定性重放与规则裁决引擎。
- 不把推演结果宣传为史实或历史预测。
- 当前阶段不创建业务实现或部署资源，除非项目所有者明确授权。

## 成功定义

- 用户能在对话中自由输入任意改写，AI 给出合理推演。
- 推演后世界状态结构化更新且不与前序矛盾，事件日志可回溯。
- 上帝面板能反映当前世界结构化状态。
- 替换模型或 Agent 框架不影响世界状态的数据语义。

## 约束与所有权

- 产品、历史内容、架构和发布的最终决策者：项目所有者。
- 仓库当前保持私有，商业模式延后到 1.0 验证后决定（D024）。
- 第三方代码、模型与史料必须记录许可证和使用条件。
- 进入实现阶段需要项目所有者明确解除 `AGENTS.md` 的限制。
