# 项目章程

> 状态：基线已确认（2026-07-12）。

## 项目身份

- 名称：Sandtable
- 方向：AI 驱动的历史沙盒
- 核心端：移动端优先 Web PWA，兼顾桌面 Web
- Agent 地基：Pi
- 当前阶段：只完善架构和项目文档，不进行具体代码实现

## 项目使命

让用户站在历史行动者的有限视角中作出选择，并通过有来源、受规则约束、可解释且可回放的模拟理解历史因果与偶然性。

## 问题陈述

- 历史内容通常是线性阅读，难以体验行动者当时的信息限制和制度约束。
- 通用 AI 角色聊天虽然自由，但容易混淆史实与生成内容，且缺乏稳定世界状态。
- 传统策略游戏有规则，但往往弱化史料来源和因果解释。
- Sandtable 将史料基线、AI 行动者、确定性裁决和世界线比较组合为统一体验。

## 目标

- 在移动端完成短时、清晰、可继续的历史推演体验。
- 让每个关键世界变化可追溯到观察、行动意图、规则与来源。
- 支持世界线暂停、重放、分支和比较。
- 使模型和部署供应商可替换，不侵入历史领域模型。

## 非目标

- 不做无约束的历史人物聊天器。
- 不把模拟结果宣传为历史预测或事实。
- MVP 不做原生 App、多人实时对战和全球历史覆盖。
- 当前阶段不创建业务代码或部署资源。

## 成功定义

- 用户能在 3 分钟内理解当前局势并完成首次有效决策。
- 用户能区分史料、模拟事件和 AI 解释。
- 关键事件解释可追溯，世界线可确定性重放。
- 移动端核心流程在主流窄屏设备上单手可完成。
- 替换模型后，行动意图协议和世界规则无需改写。

## 约束与所有权

- 产品、历史内容、架构和发布的最终决策者：项目所有者。
- 仓库当前保持私有，商业模式延后到 MVP 验证后决定。
- 第三方代码、模型与史料必须记录许可证和使用条件。
- 进入实现阶段需要项目所有者明确解除 `AGENTS.md` 的限制。
