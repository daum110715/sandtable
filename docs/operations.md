# 运行与发布

> 状态：基线已确认（2026-07-12）。本文不创建部署配置。关联决策：D016、D019、D021。

## 环境与发布

- 本地、测试、预发布、生产四级环境，数据和密钥完全隔离。
- 前端、API 和 Worker 使用独立发布单元，共享同一兼容性矩阵。
- 版本采用语义化版本；事件模式、规则和历史基线使用独立版本号。
- 数据迁移先向后兼容，再发布应用，最后清理旧结构。
- 前端静态资源可即时回滚；API/Worker 使用上一稳定容器镜像；规则和基线不得原地回滚已存在世界线。
- 模型路由和提示词采用配置发布、灰度验证和快速回退。

## SLO 基线

- 月度 API 可用性：99.9%。
- 非 AI 查询 P95：300ms 内（服务区域内）。
- 提交行动确认 P95：500ms 内返回任务标识。
- AI 认知回合：显示阶段性进度；目标 P95 30 秒内完成，超时可安全重试或降级。
- 已确认事件持久性：数据库确认后不得因 Worker 或前端故障丢失。
- 恢复目标：RPO 15 分钟，RTO 4 小时；上线前通过恢复演练验证。

## 可观测性

- 指标：请求延迟、错误率、队列深度、任务时长、重试、模型 Token/成本、工具拒绝、重放失败和投影延迟。
- 日志：结构化并带 request、user、worldline、turn、model-run 关联 ID；默认脱敏。
- Trace：覆盖浏览器请求、API、队列、Worker、Pi 模型调用和数据库事务。
- 告警：SLO 燃尽、队列阻塞、重放不一致、成本异常和安全拒绝激增。

## 容量与成本

- API 保持无状态，可水平扩展。
- Worker 按队列深度扩展，并按用户/世界线限制并发，避免同线并行写入。
- 模型调用设置单次、单用户和全局预算；低风险任务路由到低成本模型。
- PostgreSQL、Redis 和对象存储先使用托管服务，达到测量阈值后再优化。
- 单区域部署优先保证数据一致性；全球 CDN 只承载静态资源和安全缓存内容。

## 数据保留与恢复

- 原始模型会话 30 天，审计元数据 180 天；法律和安全保全例外需有记录。
- 数据库每日备份并保留点时间恢复；每季度完成一次恢复演练。
- 用户删除请求清除活动数据和对象，备份随生命周期过期且不用于日常恢复以外用途。
