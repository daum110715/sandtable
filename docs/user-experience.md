# 用户与体验

> 状态：基线已确认（2026-07-12）。关联决策：D001、D002、D003、D022、D023。

## 用户与产品形态

- 核心用户：喜爱历史、策略和架空推演的中文移动端用户。
- 次要用户：历史教育者、内容创作者和桌面端深度玩家。
- 产品形态：移动端优先 PWA；支持安装到主屏、断线恢复壳层和桌面渐进增强。
- MVP 身份：用户是拥有世界线 Root 权限的改写者；AI 控制所有历史行动者。
- 协作范围：游客可试玩，账户用于跨设备同步；只读链接可分享世界线；不做多人实时对战。

## 体验原则

- 触控目标不小于 44×44 CSS 像素。
- 核心任务采用单列布局和底部主导航，关键操作位于拇指热区。
- 一屏只突出一个主要决策，复杂信息使用渐进披露。
- 网络或模型等待必须有可理解进度，可离开页面后继续。
- 史料、模拟和 AI 推断使用稳定且不只依赖颜色的视觉标识。
- 默认支持系统深浅色、动态字号、减少动画和键盘导航。

## 核心用户旅程

1. 游客从落地页进入赤壁历史基线和原始时间线。
2. 用户选择一个模拟时刻作为改写点。
3. 用户用自然语言或结构化控件下达 Root 改写指令，例如“孙权接受招降”。
4. 系统展示结构化解释、影响范围和“非史实干预”标识，由用户确认。
5. 系统创建新世界线、写入干预事件并启动后台模拟。
6. 前端通过 SSE 展示历史行动者认知、规则裁决和世界推进阶段。
7. 用户查看结果、因果链、史料对照和行动者反应。
8. 用户继续追加改写、回到其他时刻分支，或比较多条世界线。
9. 登录后同步世界线，生成只读分享链接。

## 信息架构

- **世界**：当前局势、时间、地图摘要和待处理决策。
- **改写**：改写点、Root 指令、结构化解释、影响范围和确认。
- **时间线**：事件、因果链、史料/模拟标签和来源。
- **分支**：检查点、世界线树和差异比较。
- **档案**：行动者、制度、史料和术语。
- **我的**：账户、同步、隐私、可访问性和模型成本提示。

## 响应式策略

- 320–599px：单列、底部导航、抽屉式详情。
- 600–1023px：主内容加可选辅助面板。
- 1024px 以上：时间线、地图和详情可并列，但保持与移动端相同的信息层级。
- 不以设备 UA 判断能力，以 CSS 容器、指针类型和可用空间渐进增强。

## 可访问性与本地化

- 首发简体中文，文案和日期格式从首日支持本地化。
- 目标达到 WCAG 2.2 AA。
- 历史地名显示当时称谓，并提供现代地理说明。
- 不使用单一国家或胜负视角作为默认“正确叙事”。
