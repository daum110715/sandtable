# 用户与体验

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D002、D022、D023、D027、D028、D029、D030。

## 用户与产品形态

- 核心用户：希望自定义世界设定并探索假设后果的互动用户。
- 次要用户：教育者、内容创作者。
- 产品形态：Web 应用；对话主区与上帝面板并列布局（D022）。具体平台优先级待定（D002）。
- 1.0 身份：用户是拥有完整权限的改写者；AI 承担演员/记录员（裁判可选）角色。
- 协作范围：1.0 为单会话体验；账户同步、多人、分享留待后续（D023）。

## 入口（D027）

- **世界设定**：用户输入世界名称（可选）和设定说明（必填），定义题材、规则、人物、资源与任意初始条件。
- 不提供预设场景；创建后，设定写入初始世界状态而非仅存在于浏览器会话中。

## 核心交互 loop（D028）

```mermaid
flowchart LR
    S["定义世界设定"] --> C["与 AI 自由对话\n输入任意改写"]
    C --> D["AI 根据世界状态推演"]
    D --> P["上帝面板同步状态\n事件日志追加"]
    P --> C
```

- 定义世界设定 → 直接与 AI 自由对话，输入任意改写内容 → AI 根据当前世界状态推演结果 → 继续输入，循环往复。
- **重点**：不是选择题。用户可自由输入任何改写，不受题材或预设起点限制。

## 上帝面板（D029）

- 常驻在交互界面旁边，与对话主区并列。
- 展示当前世界的结构化状态（人物、势力、资源、地理、关系等关键要素）。
- 1.0 仅作展示，不通过它直接改写世界。
- **具体展示内容待定**，后续补充。

## 体验原则

- 对话是主操作；输入框始终可达，鼓励自由文本改写。
- 网络或模型等待必须有可理解进度，可离开页面后继续。
- 上帝面板与对话主区视觉层级清晰，面板不抢夺对话焦点。
- 默认支持系统深浅色、动态字号、减少动画和键盘导航。
- 世界设定与推演内容应有可区分的视觉标识（具体标识规范留待后续）。

## 核心用户旅程

1. 用户从入口输入世界名称（可选）和世界设定。
2. 系统创建通用初始世界状态，上帝面板展示初始结构化状态。
3. 用户在对话主区用自由文本输入改写，例如“让两个团体共享补给”。
4. 演员 Agent 读取当前世界状态，生成推演；记录员 Agent 把状态变更写回世界状态数据库。
5. 对话主区展示推演叙事；上帝面板同步更新结构化状态；事件日志追加本次改写与推演。
6. 用户继续输入下一轮改写，循环往复。
7. 用户可查看事件日志回溯历史改写-推演。

## 信息架构（1.0 最小）

- **对话主区**：自由输入改写、查看 AI 推演叙事。
- **上帝面板**：当前世界结构化状态（内容待定）。
- **事件日志/回溯**：按顺序查看每次改写与推演记录。
