# 项目路线图

路线图以验证风险为顺序，不以功能数量为顺序。

## 阶段 0：架构初始化（当前）

- 建立领域语言和项目边界；
- 明确 Pi 与 Sandtable 的责任分工；
- 记录事件驱动世界模型与 AI 权限原则；
- 选定 MVP 历史切片。

**退出条件**：MVP 选择完成，关键未决项均有负责人或决策路径。

## 阶段 1：纸面验证

- 用文档手工走通两个行动者、一个制度和一条因果链；
- 定义行动意图、裁决结果、事件和投影的示例；
- 建立史料主张与不确定性标注规范；
- 编写验收场景和反例。

**退出条件**：无需代码即可解释一次完整世界推进，且不存在角色越权或状态双写。

## 阶段 2：最小可执行切片

仅在项目所有者明确解除“禁止具体代码实现”后开始。

- 接入最小 Pi Agent 能力；
- 实现单条世界线的行动与裁决闭环；
- 验证事件重放和分支；
- 建立模型与规则版本审计。

**退出条件**：MVP 的所有必须证明项通过。

## 阶段 3：体验与内容扩展

- 增加比较、解释和时间线体验；
- 扩充同一历史切片的行动者与制度；
- 引入第二种模型 adapter，验证模型 seam；
- 建立历史内容制作和审阅流程。

## 阶段 4：多切片平台化

- 引入第二个历史切片，验证规则模块的真实 seam；
- 评估是否需要多领域上下文；
- 扩展规模、性能、协作与发布治理。

## 近期未决事项

- 首个历史切片是什么？
- 目标用户更偏教育探索、策略游戏还是研究展示？
- 世界时间采用离散步进、事件调度还是混合方式？
- 哪些高影响行动必须人工复核？
- Pi 使用同进程 SDK 还是隔离进程 RPC？
- 首阶段需要怎样的历史来源许可与引用格式？
