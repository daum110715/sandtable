# 项目路线图

> 状态：1.0 基线已确认（2026-07-12）。关联决策：D025、D032。路线图以风险验证与 1.0 闭环为优先，不以功能数量为顺序。

## 项目所有者决策

- D025 阶段划分：文档冻结 → 技术原型 → 1.0 闭环 → 面板与内容扩展 → 权限分层与平台化。
- D032 1.0 优先级：做出 1.0，不做过度设计；只跑通自由输入 → AI 推演 → 状态一致性。
- 当前阶段：技术原型，目标做出 1.0。
- 实现阶段启动批准：由项目所有者明确解除 `AGENTS.md` 的禁止代码实现限制。

## 阶段 0：方向与文档冻结（已完成）

- 确认 1.0 方向：自由改写 → AI 实时推演 → 状态一致性。
- 重写领域语言、决策清单与架构文档；标记被取代的 ADR。

**退出条件**：决策清单一致，领域语言与架构文档无未解释冲突。

## 阶段 1：技术原型（进行中）

项目所有者已于 2026-07-12 授权搭建框架；推演业务实现仍以领域协议与待定决策为前置条件。

- npm workspaces、TypeScript 严格模式与统一检查命令（已建立）。
- React + Vite 前端外壳、Fastify API、Node Worker 与共享领域协议（已建立）。
- 锁定待定决策：Agent 框架（D007）、模型路由（D014）、Agent 接入方式（D015）、数据存储技术（D016）。

**退出条件**：待定技术决策完成；领域协议与最小 schema 可在不调用模型时跑通一次改写-推演循环。

## 阶段 2：1.0 闭环

- 实现世界状态数据库与事件日志最小 schema。
- 实现演员/记录员 Agent 输入输出协议与推演编排。
- 实现对话主区 + 上帝面板并列的前端闭环。
- 跑通：自定义世界设定 → 自由输入改写 → AI 推演 → 状态写回 → 事件日志回溯。

**退出条件**：`docs/mvp.md` 的“1.0 必须跑通”全部通过；`docs/quality.md` 发布闸门通过。

## 阶段 3：面板与内容扩展

- 细化上帝面板展示内容（D029）。
- 评估世界设定模板或导入能力（不改变“无预设场景”基线）。
- 引入裁判 Agent，验证状态一致性增强。
- 史实/推演视觉标识与内容治理细化（D010、D020）。

## 阶段 4：权限分层与平台化

- 规模、性能、协作与发布治理。
