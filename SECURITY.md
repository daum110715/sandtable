# 安全政策

## 报告漏洞

如果您发现安全漏洞，请**不要**在公开的 Issue 中报告。

### 报告流程

1. **私下报告**：通过 GitHub 的 [Security Advisories](https://github.com/daum110715/sandtable/security/advisories/new) 功能提交漏洞报告
2. **邮件报告**：发送邮件至项目维护者（请在 Issue 中请求联系邮箱）
3. **包含信息**：
   - 漏洞描述
   - 复现步骤
   - 影响范围
   - 建议修复方案（如有）

### 响应时间

- **确认收到**：48 小时内
- **初步评估**：7 个工作日内
- **修复发布**：根据严重程度，30 天内

## 安全最佳实践

### 对于用户

- 始终使用最新版本
- 不要在公开场合分享 API 密钥或凭证
- 定期检查依赖更新

### 对于开发者

- 遵循最小权限原则
- 不要在代码中硬编码敏感信息
- 使用环境变量管理配置
- 定期更新依赖以修复已知漏洞

## 依赖安全

项目使用以下措施确保依赖安全：

- **依赖锁定**：使用 `package-lock.json` 锁定依赖版本
- **定期审计**：运行 `npm audit` 检查已知漏洞
- **最小依赖**：仅引入必要的依赖

## 数据安全

### 数据最小化

- 仅收集必要的用户数据
- 会话数据保留 30 天
- 审计元数据保留 180 天

### 数据存储

- 敏感数据加密存储
- 使用安全的传输协议（HTTPS）
- 遵循数据保护法规
